# Review-fix pass (code-review findings)

## What was changed

**Finding 1 (Important) — hardcoded assets-exhibit statistics.**
`ScreeningDossier.tsx`: replaced the fixed prose ("une intersection d'un seul nom entre deux
listes de six tirées parmi trente", "environ huit trades par actif sur trois ans et demi") with
a new `assetsExhibitParagraph(shared, a, b)` helper that derives every quantity from the
candidates actually rendered:
- list sizes from `a.qualified_assets.length` / `b.qualified_assets.length` (phrased as
  "deux listes de N" when equal, "une liste de N et une liste de M" when they differ),
- intersection size from `shared.length`,
- per-asset trade density from `Math.round((a.trades + b.trades) / (a.assets_go + b.assets_go))`,
  rendered as "environ N trade(s) par actif" (singular/plural handled), guarded against
  null/zero denominators (clause omitted rather than dividing by zero or printing `NaN`).
- Universe size (thirty) and history span (three and a half years) are no longer claimed
  anywhere — replaced with "sur la période mesurée" and "tout l'univers testé", neither of
  which asserts a number not present in the component's props. The core argument survives
  unchanged: the overlap is exactly what chance predicts, and the cause is that each asset
  carries very few trades.

**Finding 2 (Important) — empty-intersection falsehood.**
Same helper branches explicitly on `shared.length`:
- `0` → a "not a signal either" sentence that names no asset and asserts no shared name
  (no more `shared[0] ?? 'cet actif'` producing a nonexistent-asset claim).
- `1` → names the single shared asset by name, singular "une valeur sûre ... deux fois".
- `2+` → names all shared assets via a new `joinFr()` Oxford-style joiner ("a, b et c"),
  plural "des valeurs sûres ... deux fois", intersection size stated as "de N noms".

**Finding 3 (Minor) — sort fallback.**
`src/app/strategies/famille/[base]/page.tsx`: added a `tfRank()` helper that maps an unknown
timeframe to `TF_ORDER.length` instead of leaving `indexOf`'s `-1`, so unknown timeframes sort
after `D1`..`M5` instead of before them. `ordered` now sorts on `tfRank(a.tf) - tfRank(b.tf)`.

## Tests added (`tests/components/ScreeningDossier.test.tsx`)

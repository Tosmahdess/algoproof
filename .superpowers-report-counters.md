# Screening dashboard — Feature 1 (corpus totals) + Feature 2 (cross-timeframe coherence)

Branch: `feat/screening-dashboard`, repo `D:\code\algoproof`.

## Feature 1 — corpus totals (spec §2)

- `src/lib/screening.ts`: added `CorpusTotals` type + `corpusTotals(campaigns)`. Sums
  `n_behaviors`/`n_rejected`/`n_candidates` over `state === 'judged'` campaigns only, nulls
  treated as 0, `strategies` = distinct `base` among judged campaigns.
- `src/components/ScreeningGrid.tsx`: renders a 3-column stat block above the framing line
  (`data-testid="corpus-judged"|"corpus-rejected"|"corpus-standing"`, all sharing one
  `CORPUS_NUMBER_CLASS` string so the rejected count carries the same visual weight as judged
  and standing — no traffic-light emphasis) plus the French framing sentence
  (`data-testid="corpus-framing"`): « J'ai jugé X configurations sur Y stratégie(s). J'en ai
  rejeté Z. Il en reste W en observation. »

## Feature 2 — cross-timeframe coherence (spec §9.4)

- `src/lib/screening.ts`: added `CrossTf` type + `crossTf(campaigns, tf)`. Adjacency is position
  in `TF_ORDER` (D1↔H4↔H1↔M30↔M15↔M5); returns `null` when the timeframe itself isn't judged or
  has no standing configuration, `'isolated'` when no immediately-adjacent timeframe is judged
  with something standing, `'adjacent_coherent'` otherwise.
- `src/components/ScreeningDossier.tsx`: new `siblings: ScreeningCampaign[]` prop (required) and
  a `CrossTfNote` sub-component rendered directly under the funnel box, before the candidate
  cards/résultat-négatif/détail-indisponible branch. Isolated renders the given fragility
  sentence verbatim; adjacent_coherent names the specific neighbour(s) (singular/plural verb
  agreement, reuses the existing `joinFr` helper for two neighbours e.g. H1's H4+M30). Renders
  nothing when `crossTf` returns null.
- `src/app/strategies/famille/[base]/page.tsx`: passes `siblings={campaigns}` (the full
  per-strategy campaign list already fetched by `getStrategyDossier`) to `ScreeningDossier`.

## Constraints checked

- All new user-facing strings are French, 1st person singular where the site's author speaks
  (« J'ai jugé… », « J'en ai rejeté… »); the cross-timeframe note is an impersonal statement about
  the data (consistent with the rest of the dossier's prose) — no "on"/"nous" anywhere.
- No traffic-light colouring; the three corpus numbers and the two cross-timeframe prose branches
  use plain text-only styling.
- No internal vocabulary (`GO_PAPER`/`MARGINAL`/`NO_GO`/`strong`) or bare "survivants" rendered —
  grepped both new component files, only a hit is a pre-existing code comment (not on-screen).
- No parameter key/value or per-asset PF rendered (untouched code paths).
- Both edited components remain server components: no `'use client'`, no hooks added.
